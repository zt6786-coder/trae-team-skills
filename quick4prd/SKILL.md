# quick4prd - 专业文档全流程生成技能

## 核心定位
15年阿里资深PM实战沉淀，覆盖 PRD、产品方案、汇报材料、标书、可研报告、白皮书、项目申报书等 95% 专业文档场景。内置「结构化大纲 → 逐节点深度展开 → 格式自适配」三段式工作流，支持预算驱动篇幅动态控制。

## 适用场景
- **产品经理**：PRD、产品方案、迭代规划文档
- **解决方案架构师**：技术方案、架构设计文档、POC方案
- **高管/汇报**：季度汇报、年度规划、董事会材料、OKR复盘
- **投标/申报**：招标响应文件、可研报告、项目申报书、政府专项申报
- **咨询/交付**：交付方案、实施计划、验收报告、白皮书

## 三段式工作流

### 第一阶段：结构化大纲生成
**输入**：业务背景、核心目标、受众画像、篇幅预算（页数/字数/预算金额）
**输出**：标准化文档骨架（含章节层级、关键节点、预估篇幅分配）
**关键动作**：
- 基于文档类型自动匹配标准模板库（PRD/方案/汇报/标书/可研/白皮书/申报书）
- 预算驱动篇幅分配引擎：总预算 → 章节权重 → 单节点字数上限
- 关键决策略读者视角重构：按受众（技术/业务/高管/评委）差异化调整大纲深度

### 第二阶段：逐节点深度展开
**输入**：确认后的大纲、参考资料（可选）、企业知识库（可选）
**输出**：全文初稿（Markdown格式，含图表占位、数据引用标记）
**关键动作**：
- 节点级Prompt工程：每章节独立Prompt，保证专业深度与术语准确
- 证据链自动构建：引用数据源、竞品对比、行业标准、历史案例
- 逻辑自洽性校验：前后章节术语统一、数据口径一致、论证闭环
- 并行化生成：独立章节可并行，大幅缩短长文档生成耗时

### 第三阶段：格式自适配与交付
**输入**：全文初稿、目标格式（Word/PDF/PPT/Markdown/在线文档）
**输出**：成品文档（含目录、页眉页脚、编号规范、图表编号、交叉引用）
**关键动作**：
- 多格式渲染：docx、pdf、pptx、md
- 规范化排版：国家标准/企业模板/投标要求（字体、行距、页边距、编号体系）
- 质量门禁：AI味检测（接humanizer-zh）、敏感词扫描、格式合规性自检
- 版本管理：生成变更记录、差异对比、审阅批注模式

## 预算驱动篇幅控制引擎
| 文档类型 | 典型预算范围 | 篇幅映射规则 | 关键控制点 |
|---------|-------------|-------------|-----------|
| PRD | 5-30页 | 核心功能清单占40%，非功能/交互/数据各15% | 功能点粒度可配置 |
| 产品方案 | 10-50页 | 架构设计30%，实施路径25%，风险/成本/收益各15% | 技术深度分级 |
| 汇报材料 | 15-40页 | 结论先行（1页），现状/问题/对策/规划四象限 | 高管阅读节奏 |
| 标书/可研 | 50-200页 | 按评分项权重分配，商务/技术/服务/报价分卷 | 评分点逐条响应 |
| 白皮书 | 30-80页 | 行业洞察30%，技术主张25%，案例/生态/路线图 | 思想领导力调性 |
| 申报书 | 20-60页 | 创新性/可行性/效益/团队四维度，政策关键词植入 | 评审专家视角 |

## 协作接口
- **上游**：接收 research-guide 产出的调研报告/竞品分析/数据支撑；接收 data-analysis 产出的测算模型/图表
- **下游**：输出标准化 Markdown 供 humanizer-zh 去AI味；输出 docx/pdf 供 pptx/html-report 二次排版
- **并行**：可与 bidding-analyzer 协同（标书技术卷+商务卷分工）

- 可选输出：docx、pdf、pptx（需指定模板或使用内置规范模板）
- 质量自检报告：篇幅达成率、关键节点覆盖率、术语一致性、引用完整性
