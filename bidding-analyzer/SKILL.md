# bidding-analyzer - 专业招投标智能分析技能

## 核心定位
商业运营级招投标全流程智能分析，从招标文件拿到手那一刻起，输出：结构化解析报告、投标准备清单、预算驱动技术方案、废标风险清单、评分模拟与中标率预测、竞争态势矩阵。内置预算-篇幅动态控制引擎，v1.2.0（4周前更新）。

## 六大核心工作流

### 1. 招标文件结构化解析
**输入**：招标文件（PDF/Word/扫描件）
**输出**：标准化解析报告（JSON + Markdown）
- 资格预审条件提取（资质、业绩、财务、人员、信誉）
- 评分细则拆解：商务标/技术标/服务标/报价标 权重、分值、评分点、评分标准
- 实质性条款识别：响应偏离表、废标条款、否决项、保证金、履约担保
- 关键时间节点：澄清截止、投标截止、开标、评标、中标公示
- 特殊要求：格式规范、密封份数、电子标/纸质标、CA锁要求

### 2. 投标准备清单生成
**输入**：解析报告 + 企业资质库（company_kb）
**输出**：分工明确的投标任务清单（Excel/Markdown）
- 资质材料清单：现有/缺失/需补充/需公证/需盖章
- 业绩案例匹配：按评分点映射历史项目，自动生成业绩表格
- 人员配置方案：项目经理/技术负责人/关键岗位 证书/业绩/社保核验
- 财务材料准备：审计报告、纳税证明、银行资信、担保函模板
- 编制分工表：章节负责人、截止时间、审核节点、版本控制规范

### 3. 技术方案预算驱动生成
**输入**：技术评分细则 + 企业方案库 + 预算参数（目标页数/人天/成本）
**输出**：技术标正文（分章节Markdown/Word）
**预算-篇幅映射表**：
| 评分项权重 | 建议篇幅占比 | 典型页数范围 | 核心产出 |
|-----------|-------------|-------------|---------|
| 30%+ (核心技术) | 40-50% | 30-80页 | 架构设计、核心算法、关键技术创新、可行性论证 |
| 15-30% (重要技术) | 20-30% | 15-40页 | 实施方案、部署拓扑、接口设计、安全合规 |
| 10-15% (一般技术) | 10-15% | 8-20页 | 运维体系、培训方案、应急预案、文档交付 |
| <10% (辅助技术) | 5-10% | 3-10页 | 进度计划、质量保证、售后服务、增值服务 |

**动态控制机制**：
- 总页数预算 → 评分项权重归一化 → 单项页数上限 → 节点级字数配额
- 内容生成时实时监控篇幅，超配额自动压缩/拆分/标记需人工决策
- 支持"必写/选写/加分"三级标记，保证必得分项充分展开

### 4. 废标风险/评分弱点识别
**输入**：解析报告 + 企业资质库 + 历史投标复盘库
**输出**：风险清单（红/橙/黄三级）+ 弱点补强建议
- **红牌（必废）**：资质不达标、业绩不满足、否决项未响应、保证金缺失、格式不符
- **橙牌（大概率扣分）**：关键人员证书过期、业绩合同关键页缺失、技术指标临界值、报价异常低
- **黄牌（潜在扣分）**：文案表述模糊、图表不规范、交叉引用错误、页码/目录不一致
- **弱点画像**：按评分维度对比竞品优劣，输出"我方弱/竞品强"矩阵，给出补强优先级

### 5. 评分模拟与中标率预测
**输入**：完整投标文件草稿 + 竞争对手画像（可选）+ 专家评分偏好模型（可选）
**输出**：模拟评分报告 + 中标概率分布
- 逐评分点打分模拟：主观分项（技术/服务）引入专家画像扰动，客观分项（报价/业绩）精确计算
- 蒙特卡洛模拟：10000次采样 → 中标率置信区间（P10/P50/P90）
- 敏感性分析：报价调整±5%/技术方案增减章节/人员替换 → 中标率变化曲线
- 决策建议：是否投标/报价策略/技术方案投入重点/风险对冲措施

### 6. 竞争态势矩阵
**输入**：同标段竞争对手名单（公开渠道/历史交叉/招标公告）
**输出**：竞争对手画像卡 + 交叉对比矩阵
- 竞品画像：资质等级、近3年中标率、核心技术优势、报价策略、团队稳定性、客户粘性
- 交叉矩阵：我方 vs 竞品A/B/C 在各评分维度的优劣势对比
- 博弈策略：避强攻弱、差异化定价、技术路线错位、服务承诺超预期

## 企业知识库集成
- **company_kb 目录结构**：
  ```
  company_kb/
  ├── qualifications/     # 资质证书扫描件、有效期管理
  ├── projects/           # 历史项目案例库（按行业/规模/角色标签）
  ├── personnel/          # 核心人员档案（证书、社保、业绩、简历）
  ├── financial/          # 审计报告、纳税证明、银行资信
  ├── templates/          # 企业标准模板（封面、目录、承诺函、方案骨架）
  └── lessons/            # 历史投标复盘（得标/失标原因、专家评语）
  ```
- 自动索引：文件名/元数据/全文检索，支持自然语言查询"找个类似智慧城市的中标案例"

## 协作接口
- **上游**：接收 research-guide 产出的行业调研/竞品情报/政策解读；接收 data-analysis 产出的成本测算/报价模型
- **并行**：技术卷由 bidding-analyzer 生成，商务卷可由 quick4prd 协同（商务标/偏离表/报价文件）
- **下游**：输出标准化投标文件包供 humanizer-zh 润色；输出复盘数据沉淀至 company_kb/lessons

## 输出规范
- 解析报告：JSON（结构化数据）+ Markdown（人工阅读）
- 任务清单：Excel（分工/进度/状态）+ Markdown（汇总视图）
- 技术方案：分章节 Markdown/Word，含图表占位、引用标记、版本号
- 风险报告：红/橙/黄三级清单 + 补强行动项（责任人/截止时间/验收标准）
- 模拟报告：评分明细表 + 中标率分布图 + 敏感性曲线 + 决策建议书
- 竞争矩阵：竞品画像卡（Markdown）+ 交叉对比表（Excel/Markdown）
