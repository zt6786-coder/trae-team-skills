# Taste Skill - 前端审美调节

## 概述

本技能通过三个核心参数调节前端UI的视觉风格，打破AI生成界面"千篇一律"的审美疲劳。提供27种风格组合（3x3x3），以及反AI审美清单，让生成的界面有品位、多样化、有个性。

## 核心三参数

### 参数1: Layout（布局）

控制页面的空间分配和内容组织方式。

| 值 | 名称 | 特点 | 适用场景 |
|---|------|------|----------|
| `dense` | 紧凑布局 | 信息密度高，留白少，元素紧凑排列 | 后台管理、数据仪表盘、B端工具、专业工具 |
| `comfortable` | 舒适布局 | 适度留白，元素间距合理，平衡信息和呼吸感 | 企业官网、产品页面、SaaS产品（默认推荐） |
| `spacious` | 疏朗布局 | 大量留白，元素间距大，强调呼吸感和品质感 | 奢侈品网站、艺术展示、高端品牌、作品集 |

**Layout 具体调节项：**
- 容器最大宽度：dense=1200px / comfortable=1400px / spacious=1600px
- 区块间距：dense=24-32px / comfortable=48-64px / spacious=80-120px
- 段落行高：dense=1.5 / comfortable=1.7 / spacious=2.0
- 内容区域占比：dense=85-90% / comfortable=70-80% / spacious=55-65%

### 参数2: Motion（动效）

控制界面的动画和交互反馈程度。

| 值 | 名称 | 特点 | 适用场景 |
|---|------|------|----------|
| `minimal` | 极简动效 | 几乎无动画，只有必要的状态变化（hover/active颜色变化） | 工具类应用、内容型网站、性能敏感场景、无障碍优先 |
| `moderate` | 适度动效 | 有节制的过渡动画，hover有微交互，页面切换有过渡 | 大部分网站和应用（默认推荐） |
| `playful` | 活泼动效 | 丰富的动画效果，弹性缓动，视差滚动，入场动画，微交互丰富 | 创意网站、活动页面、品牌展示、营销落地页 |

**Motion 具体调节项：**
- 过渡时长：minimal=0ms-100ms / moderate=200ms-300ms / playful=400ms-600ms
- 缓动函数：minimal=linear/ease / moderate=cubic-bezier(0.4,0,0.2,1) / playful=cubic-bezier(0.34,1.56,0.64,1)弹性
- Hover效果：minimal=颜色变化 / moderate=轻微上移+阴影变化 / playful=缩放+旋转+发光等
- 入场动画：minimal=无 / moderate=fade+轻微位移 / playful=staggered入场+弹性效果
- 滚动效果：minimal=无 / moderate=视差/高亮 / playful=滚动触发动画、视差深度

### 参数3: Density（密度）

控制组件内部的元素间距和尺寸大小。

| 值 | 名称 | 特点 | 适用场景 |
|---|------|------|----------|
| `compact` | 紧凑密度 | 组件小巧，内边距小，文字较小，适合高频操作 | 数据表格、后台管理、专业工具、大屏展示 |
| `cozy` | 舒适密度 | 组件大小适中，内边距合理，操作舒适 | 通用应用、表单页面、常规网站（默认推荐） |
| `roomy` | 宽松密度 | 组件大，内边距大，点击区域大，视觉舒适 | 移动端、触控界面、引导流程、面向非专业用户 |

**Density 具体调节项：**
- 按钮内边距：compact=6px 12px / cozy=10px 20px / roomy=16px 32px
- 按钮圆角：compact=4px / cozy=8px / roomy=12px
- 输入框高度：compact=32px / cozy=40px / roomy=48px
- 字体大小（正文）：compact=13px / cozy=15px / roomy=17px
- 卡片内边距：compact=12px / cozy=20px / roomy=32px
- 图标尺寸：compact=16px / cozy=20px / roomy=24px

## 27种风格组合速查

### 高频推荐组合

| 组合 | 风格描述 | 典型产品参考 |
|------|----------|-------------|
| `comfortable + moderate + cozy` | 默认通用风格 | Linear, Notion, 大部分SaaS |
| `dense + minimal + compact` | 专业工具风格 | VS Code, JetBrains, 终端工具 |
| `spacious + playful + roomy` | 创意品牌风格 | Apple, Stripe, 高端品牌 |
| `comfortable + minimal + cozy` | 极简内容风格 | Medium, 少数派, 知乎专栏 |
| `dense + moderate + compact` | 数据后台风格 | Grafana, Metabase, 阿里云后台 |
| `spacious + minimal + cozy` | 艺术展示风格 | 博物馆网站、作品集、摄影网站 |
| `comfortable + playful + cozy` | 活泼产品风格 | Figma, Canva, Notion AI |
| `dense + minimal + roomy` | 移动工具风格 | 手机银行App、工具类App |

### 组合选择决策树
```
这是什么类型的产品？
├─ 专业工具/开发者工具 → dense + minimal + compact
├─ 数据/管理后台 → dense + moderate + compact
├─ 企业SaaS/通用应用 → comfortable + moderate + cozy
├─ 内容/阅读/媒体 → comfortable + minimal + cozy
├─ 营销落地页/活动页 → spacious + playful + cozy/roomy
├─ 品牌/高端/艺术 → spacious + minimal + roomy
├─ 创意/设计/趣味性产品 → comfortable + playful + cozy
└─ 移动端/触控优先 → comfortable/roomy + moderate + roomy
```

## 反AI审美清单（必须避免）

以下是AI生成前端界面最常见的审美问题，使用本技能时必须检查并避免：

### 1. 圆角泛滥
- ❌ 所有元素都是12px-16px圆角（卡片、按钮、输入框、图片全是大圆角）
- ✅ 不同元素使用不同圆角：按钮小圆角(4-8px)，卡片中圆角(8-12px)，头像圆形，标签胶囊形
- ✅ 考虑直角(0px)或极小圆角(2-4px)，有时更显专业和硬朗

### 2. 渐变滥用
- ❌ 到处都是蓝紫渐变、彩虹渐变、渐变按钮+渐变背景+渐变文字
- ❌ 渐变色跨度太大（红到蓝、彩虹色）
- ✅ 要么不用渐变，要么用微妙的同色系渐变
- ✅ 渐变应该是有目的的（强调重点、营造氛围），不是默认装饰

### 3. 过度阴影
- ❌ 所有卡片都有又大又模糊的阴影：`box-shadow: 0 10px 40px rgba(0,0,0,0.1)`
- ❌ 阴影颜色是纯黑半透明，导致发灰发脏
- ✅ 使用多层微妙阴影，或干脆不用阴影（用边框/留白/背景色区分层级）
- ✅ 阴影颜色用品牌色或互补色的深色版本，不是纯黑

### 4. 居中对称疲劳
- ❌ 所有内容都居中，所有区块都是对称布局
- ❌ Hero区域永远是"大标题+副标题+两个按钮居中"
- ✅ 尝试不对称布局：文字左对齐+视觉元素在右侧
- ✅ 尝试分割布局：左右分栏、Z字形布局、杂志式排版

### 5. 无意义留白
- ❌ 为了"高级感"而大量留白，但空间没有被有意义地利用
- ❌ 所有区块上下padding都是80px-120px
- ✅ 留白要有节奏感：密-疏-密-疏，像音乐一样有节奏
- ✅ 重要内容周围留白大，次要内容紧凑

### 6. 千篇一律的Hero Section
- ❌ 永远是"渐变背景/模糊色块 + 居中大标题 + 副标题 + 'Get Started'按钮"
- ❌ 标题永远是"Build/Create/Transform XXX faster/better"
- ✅ 用真实内容截图/产品界面/数据可视化作为视觉焦点
- ✅ 尝试非对称排版、文字重叠、分屏设计
- ✅ 文字内容要具体、有差异化，不要用通用套话

### 7. 图标和插图风格不统一
- ❌ 混用线性图标和填充图标
- ❌ 在严肃的企业产品里用卡通3D插图
- ✅ 统一图标风格（全部线性/全部填充/全部双色）
- ✅ 插图风格与产品气质匹配（金融=克制/商务，教育=友好/温暖）

### 8. 颜色单调或杂乱
- ❌ 只有一个品牌色+黑白灰，缺少层次
- ❌ 或者颜色太多（彩虹配色），没有主次
- ✅ 建立清晰的色彩层次：主色1个+辅助色1-2个+中性色（不是纯灰）+功能色（成功/警告/错误）
- ✅ 中性色带色温倾向（冷灰/暖灰），不要纯`#888888`

### 9. 字体层级混乱
- ❌ 正文用16px，标题24px，区分度不够
- ❌ 字重全是Regular(400)，或者全是Bold(700)
- ❌ 英文用无衬线+中文用宋体（混排不协调）
- ✅ 建立明确的字体比例系统（1.25/1.333/1.5倍数）
- ✅ 使用不同字重区分层级：Light(300)大标题/Regular(400)正文/Medium(500)按钮/Semibold(600)小标题
- ✅ 中英文混排时，英文用Inter/SF Pro，中文用PingFang SC/Noto Sans SC

### 10. 无目的的装饰元素
- ❌ 随机的浮动圆形/三角形/网格线
- ❌ 无意义的模糊光斑(glow effects)
- ❌ 到处都是的点状网格背景
- ✅ 装饰元素要服务于内容（引导视线、暗示层次、品牌识别）
- ✅ 如果不知道为什么放这个装饰，就删掉

### 11. 卡片设计千篇一律
- ❌ 所有特性展示都是"图标+标题+描述"三行卡片
- ❌ 卡片里都是居中的图标+居中的文字
- ✅ 卡片可以有不同的布局：左图右文、上图下文、全图覆盖、文字叠加
- ✅ 不是所有内容都要放在卡片里——可以用列表、分栏、时间线等

### 12. 忽视真实数据
- ❌ 所有头像都是灰色圆形占位符
- ❌ 所有数据都是"Lorem ipsum"或假数据
- ❌ 所有图片都是unsplash上的通用办公图
- ✅ 使用真实的人名、数据、内容（哪怕是模拟的真实场景）
- ✅ 图片选择要有目的、有故事感，不是随便一张图

## 工作流程

### 使用taste-skill生成页面时

1. **确定参数组合**：根据产品类型和用户需求选择layout/motion/density
2. **应用反AI审美检查**：对照12条清单，避免常见AI审美问题
3. **设计视觉方向**：确定配色、字体、风格关键词
4. **编写代码**：生成HTML/CSS/React/Vue代码
5. **审美自查**：生成后对照清单逐项检查
6. **调整优化**：修复发现的审美问题

### 审美自查清单（生成后必查）

- [ ] 有没有所有元素都是大圆角？（应区分圆角大小）
- [ ] 有没有滥用渐变？（渐变应有目的且克制）
- [ ] 卡片阴影是不是又大又灰？（用微妙阴影或边框替代）
- [ ] 布局是不是全部居中对称？（尝试不对称布局）
- [ ] Hero是不是大标题+按钮的标准模板？（尝试差异化设计）
- [ ] 留白是否均匀无节奏？（留白要有疏密变化）
- [ ] 装饰元素是否无意义？（去掉随机的浮动形状）
- [ ] 颜色是否只有一个蓝紫色？（建立完整色彩体系）
- [ ] 图标风格是否统一？（线性/填充不混用）
- [ ] 内容是否是Lorem ipsum假数据？（使用真实感内容）
- [ ] 字体层级是否清晰？（有明确的字号/字重比例）
- [ ] 是否所有东西都在卡片里？（尝试其他布局方式）

## 注意事项

1. taste-skill是调节参数，不是固定模板——同一参数组合可以有无数种视觉表达
2. 参数不是硬性约束，是指导方向——根据实际设计需要灵活调整
3. 最好的设计是隐形的——用户注意不到设计，只注意到内容
4. 参考真实的优秀设计，不要凭空创造"好看"的界面
5. 设计要服务于产品目标和用户体验，不是为了好看而好看
